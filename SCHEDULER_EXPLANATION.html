📧 Lead Outreach Scheduler - What I'll Build

The Big Picture

I'm building an automated system that:

  • Reads a list of leads from a CSV file
  • Contacts each lead one at a time through 4 different platforms
  • Keeps track of who we've already messaged
  • Never bothers people who have already replied
  • Updates the CSV file with when we contacted each person

The 4 Contact Platforms (In Order)

For each lead, we try these platforms in this exact order:

  1. 📧 Email
  2. 👍 Facebook
  3. 📸 Instagram
  4. 💬 WhatsApp

We stop at the first platform where we can successfully send a message.

The 4 Smart Rules

Rule #1: Daily Limit

Don't send more than 10 messages per platform per day.

If we hit 10 emails today, skip emailing for the rest of today's run.

Rule #2: Cooldown Period

Don't contact the same person more than once every 3 days.

If we last messaged someone 2 days ago, skip them today.

Rule #3: Respect Responses

If someone replies, stop contacting them.

A human marks "YES" in the `responded` column. Once set, we never contact that person again.

Rule #4: Try Next Platform

Move to the next platform if the previous one was already used.

If we already emailed them, try Facebook next. If Facebook is done, try Instagram. And so on.

What Happens Each Day

The scheduler runs once per day (usually early morning) and:

  1. Reads all the leads from the CSV file
  2. Goes through each lead one by one
  3. Checks all 4 rules for that lead
  4. Sends a message if it should
  5. Writes the date we sent it to the CSV
  6. Moves to the next lead
  7. Saves the updated CSV when done

The CSV Columns We Track

Contact Info (humans fill these):
• name
• email
• phone
• facebook
• instagram
When We Contacted Them (scheduler fills these):
• email_sent (date)
• facebook_sent (date)
• instagram_sent (date)
• whatsapp_sent (date)
Status Flags (humans set "YES"):
• responded (YES = stop contacting)
• restart_needed (scheduler writes YES when all 4 platforms are exhausted)

Special Cases We Handle

  • Missing data: If someone has no email, no Facebook, no Instagram, and no phone → skip them completely
  • Send fails: If a message doesn't send, we don't write the date → we'll try again tomorrow
  • Bad dates: If someone has a weird date in the CSV → treat it as blank, try again
  • All platforms used: If all 4 platforms have been tried and they haven't replied → mark them as needing manual restart

What Gets Logged

After each run, we log:

  • When the run happened
  • How many messages we sent on each platform
  • Which people we skipped (and why)
  • Which people need manual restart
  • Any sending failures with names
